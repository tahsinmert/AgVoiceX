# Support

Use GitHub Issues for reproducible bugs, feature requests, and documentation gaps.

Before opening an issue:

- Search existing issues.
- Run `bash scripts/dev-check.sh` when reporting local stack problems.
- Include backend logs with secrets removed.
- Include the local provider and model name for AI runtime issues.
- Include browser, operating system, and console errors for frontend issues.

Commercial support, hosted deployments, and managed operations are not provided by this repository unless separately arranged with the maintainer.
